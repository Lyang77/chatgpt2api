# 图片轮询超时：直接失败

## 目标

`image_poll_timeout_secs` 是一次图片任务获取结果的唯一轮询预算。轮询预算耗尽后，当前图片任务立即失败，不继续占用账号槽、后台线程或上游轮询资源。

## 行为

1. 任务先完成账号获取和上游图片请求；当需要等待图片结果时，进入轮询。
2. 轮询的最大等待时间始终使用 `config.image_poll_timeout_secs`，不因上游先返回文本或异步状态而提升为 300 秒。
3. 抛出 `ImagePollTimeoutError` 后，当前图片直接失败；不换账号重试，也不执行后备轮询。
4. 网络/TLS 等非图片轮询超时的现有重试策略不在本次变更范围内。
5. 调用日志仍记录任务的全流程耗时；它可能略大于轮询配置，因为账号获取、上游请求和已发出的 HTTP 请求不属于可中断的轮询等待。

## 测试

- 回归测试覆盖：图片轮询超时时不再次获取账号、不发起第二次生成尝试，并将原始 `ImagePollTimeoutError` 交给上层失败处理。
- 覆盖文本回复路径：它向轮询函数传递配置值，不再强制 300 秒。
