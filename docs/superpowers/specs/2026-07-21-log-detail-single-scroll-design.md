# 日志详情单滚动设计

## 目标

日志详情弹窗在长文本场景下只显示一个纵向滚动条，避免弹窗正文与文本内容区同时滚动。

## 当前问题

`web/src/app/logs/page.tsx` 的日志详情弹窗有两层滚动容器：

- 弹窗正文使用 `overflow-y-auto`。
- 结果内容、请求内容和原始数据的 `<pre>` 同时使用 `max-h-*` 与 `overflow-auto`。

长文本超过 `<pre>` 的最大高度后，两层容器都会产生纵向滚动条。

## 方案

保留弹窗正文作为唯一滚动容器。移除三个 `<pre>` 上的 `max-h-*` 和 `overflow-auto`，保留自动换行、断词、背景、间距和字体样式。

不调整弹窗尺寸，不新增折叠交互，不修改日志内容、接口或后端存储。

## 验收标准

- 结果内容、请求内容和原始数据三个标签页均不再创建内部滚动区域。
- 长内容通过弹窗正文右侧的唯一滚动条查看。
- 文本继续自动换行，横向不溢出。
- 前端静态检查与构建通过。
- 通过实际页面截图确认日志详情弹窗只显示一个纵向滚动条。

## 测试策略

新增一个针对日志详情样式契约的前端测试，先证明当前三个 `<pre>` 仍包含内部滚动类，再修改页面使测试通过。随后运行相关前端测试、静态检查和生产构建，并在本地页面做视觉验证。
