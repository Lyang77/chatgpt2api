# 进行中调用日志实时耗时

## 目标

日志管理页中的 `running` 调用任务应持续显示从 `started_at` 到当前时刻的耗时，不新增后台请求或日志库写入。

## 设计

1. 日志管理前端在当前页含 `running` 调用记录时每秒刷新本地时钟；耗时由 `started_at` 与该本地时钟计算。
2. 页面不新增任何定时 API 请求、SQLite 写入或 JSON 写入。
3. 不改变已结束日志的持久化 `duration_ms`，也不影响筛选、分页、详情、删除或停止逻辑。

## 验证

- 类型检查证明前端变更有效；代码审查确认不存在 `fetchSystemLogs` 的定时调用。
