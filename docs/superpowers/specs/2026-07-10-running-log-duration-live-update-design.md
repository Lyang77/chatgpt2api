# 进行中调用日志实时耗时

## 目标

日志管理页中的 `running` 调用任务应持续显示从 `started_at` 到当前时刻的耗时，并定期同步后台状态、阶段和执行账号，不以每秒写入日志库的方式实现。

## 设计

1. `LogService.list()` 与 `get_by_id()` 返回 `running` 调用日志时，在返回副本中计算 `duration_ms = now - started_at`；不更新 SQLite 或 JSON 存储。
2. 日志管理前端在当前页含 `running` 调用记录时每秒刷新本地时钟，使耗时标签更新；每 5 秒重新请求当前列表以获取终态、阶段和账号变化。
3. 不改变已结束日志的持久化 `duration_ms`，也不影响筛选、分页、详情、删除或停止逻辑。

## 验证

- 后端单元测试证明：运行中记录的返回耗时由当前时间计算，存储值不被写回；终态记录保持原值。
- 前端测试证明：存在运行中记录时建立秒级显示刷新和 5 秒列表同步；无运行中记录时不轮询。
